I've drafted the Diabetes Advanced Care — Tier 3 pathway. Here's what I've built:

Eligibility: T2DM (E11.x) + HbA1c 8.0–10.0% OR PDC <80% OR active complication OR ≥1 DM-related ER visit in 12mo

Care team: Dedicated RN (monthly), Clinical Pharmacist (monthly), RD/CDCES (q6wk), Endocrinologist (quarterly), BH specialist (if PHQ-9 ≥10)

Protocol steps:

  1. Chart review & eligibility verification
  2. Patient contact & enrollment
  3. AI careplan generation
  4. Adherence barrier assessment (before medication escalation)
  5. Monthly clinical review (recurring)
  6. Specialist referrals (ophthalmology, nephrology per gaps)
  7. Medication intensification (if adherence resolved + HbA1c still elevated)
  8. Quarterly outcome review & re-cohortisation check

Escalation triggers: HbA1c >10% → Tier 4 auto-promote. DKA event → Tier 4 immediate. PDC <70% for 60d → pharmacist escalation.

Three Modes

✨ AI Builder (shown above)

Describe the pathway in plain language or voice. AI drafts the full pathway, asks one clarification, renders it as a flow diagram. Fastest path from idea to pathway.

⬡ Visual Canvas

Drag-and-drop editing. Component library on the left: eligibility blocks, action blocks (outreach, lab order, referral, escalation), logic blocks (if/else, wait, timer), message templates. Connect with arrows on a canvas.

⚙ Configuration

Click any block to open its config drawer. Set channel (WhatsApp/SMS/call), language, message template, cadence for outreach blocks. Set test type, frequency, threshold triggers for lab blocks. Set escalation rules.

Publish & Rollout

  • Draft → Review → Published lifecycle. Pathways can be versioned.
  • Auto-rollout: When published, the cohortisation engine automatically assigns eligible patients to the pathway. No manual enrollment.
  • Pathway is generic: The builder doesn't know about diabetes specifically. It works with blocks: eligibility criteria, actions, logic gates, escalation triggers, recurring schedules. Diabetes is just the first pathway built through it.
