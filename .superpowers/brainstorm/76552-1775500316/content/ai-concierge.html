How it connects

  • Design time (Pathway Builder): The Outreach block config defines the concierge sequence — steps, channels, templates, timeouts, escalation rules, safety keywords. This is the equivalent of the concierge template registry + triage config from the spec.
  • Runtime (Communications → Orchestration): When a patient hits a pathway step that has an Outreach block, the orchestration engine fires the sequence. The table shows every active sequence with its state machine position (Sent → Awaiting → Confirmed / Escalated).
  • Escalation → Thread view: Clicking "View" on an escalated row opens the thread view (the earlier Communications mockup) with full message history and patient context panel. Care manager takes over from there.
  • Escalation → Command Center: Escalated sequences also surface as action items in the Command Center's AI Action Queue.
  • State machine: Idle → WA Sent → Awaiting → (Confirmed / Needs Review / Follow-up Sent → Awaiting → Escalated). Append-only action log per the concierge spec pattern.
  • Safety keywords: Patient replies containing emergency/cost/confirmation keywords get routed immediately — before the timeout. LLM classifies ambiguous replies.
