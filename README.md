# Managed Care Platform

Production managed care program platform for healthcare payors. Full-stack:

- **Frontend** — Next.js 15 (App Router) · React 19 · TypeScript · Tailwind 4 · Zustand
- **Backend**  — FastAPI · SQLAlchemy 2.0 (async) · PostgreSQL 16 · Alembic
- **AI**       — Google Gemini 2.5 Flash (streaming SSE for pathways, summaries, insights)

---

## Quick start (Docker — recommended)

You need **Docker Desktop** (or Docker Engine + the `compose` plugin) and a **Gemini API key**.

```bash
git clone https://github.com/devops-tatvacare/managed-care-platform.git
cd managed-care-platform

# 1. One-shot setup: checks prereqs, creates .env, builds images
./scripts/setup.sh

# 2. Add your Gemini key to .env (if you didn't already)
#    GEMINI_API_KEY=...   ← https://aistudio.google.com/apikey

# 3. Start the stack
docker compose up
```

Then open:

| Surface       | URL                            |
| ------------- | ------------------------------ |
| App           | http://localhost:3000          |
| API docs      | http://localhost:8000/docs     |
| Postgres      | `localhost:5432` (postgres/postgres, db `managed_care_platform`) |

To stop: `Ctrl+C`, then `docker compose down`. Data persists in the `pgdata` volume.
To wipe and start clean: `docker compose down -v`.

---

## What `setup.sh` does

1. Verifies `docker` + `docker compose` are installed and the daemon is running.
2. Copies `.env.example` → `.env` (only if `.env` doesn't already exist).
3. Generates a strong `JWT_SECRET` if the placeholder is still in `.env`.
4. Warns if `GEMINI_API_KEY` is empty.
5. Runs `docker compose build` so first `up` is fast.

It is idempotent — safe to re-run.

## Environment variables

See `.env.example` for the full list. Required to run:

| Variable             | Purpose                                          |
| -------------------- | ------------------------------------------------ |
| `GEMINI_API_KEY`     | Google AI Studio key for all LLM features        |
| `LLM_DEFAULT_MODEL`  | Default Gemini model (e.g. `gemini-2.5-flash`)   |
| `DATABASE_URL`       | Async Postgres URL (`postgresql+asyncpg://...`)  |
| `JWT_SECRET`         | 32-byte hex string used to sign JWTs             |
| `CORS_ORIGINS`       | JSON array of allowed origins                    |

In Docker, `DATABASE_URL` and `CORS_ORIGINS` are overridden by `docker-compose.yml` to point at the `db` service.

## Local dev (without Docker)

Use this only if you want hot-reload across both frontend and backend.

**Prereqs:** Node 20+, pnpm 9+, Python 3.11+, PostgreSQL 16.

```bash
# Postgres (skip if you already have one)
docker run -d --name mcp-pg -p 5432:5432 \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=managed_care_platform \
  postgres:16-alpine

# Frontend deps
pnpm install

# Backend venv + deps
pnpm setup:backend

# DB migrations
pnpm migrate

# Run frontend + backend together
pnpm dev:all
```

Common scripts:

| Command                  | What it does                            |
| ------------------------ | --------------------------------------- |
| `pnpm dev`               | Frontend only (Next.js, port 3000)      |
| `pnpm dev:backend`       | Backend only (Uvicorn reload, port 8000)|
| `pnpm dev:all`           | Both, concurrently                      |
| `pnpm build`             | Production frontend build               |
| `pnpm lint`              | ESLint                                  |
| `pnpm migrate`           | `alembic upgrade head`                  |
| `pnpm migrate:create -- "msg"` | New autogenerated migration       |
| `pnpm migrate:rollback`  | Downgrade one migration                 |

├── src/                      # Next.js app (App Router)

- `/dashboard/outcomes` — Clinical, HEDIS, financial metrics

## Troubleshooting

| Symptom                                              | Fix                                                                 |
| ---------------------------------------------------- | ------------------------------------------------------------------- |
| `Cannot connect to the Docker daemon`                | Start Docker Desktop and re-run.                                    |
| Backend container exits with `asyncpg` connect error | Check `DATABASE_URL` uses `postgresql+asyncpg://` (not plain).      |
| AI endpoints return 500                              | `GEMINI_API_KEY` missing/invalid in `.env`.                         |
| Port 3000 / 8000 / 5432 already in use               | Stop the conflicting process or change the port in `docker-compose.yml`. |
| Migrations out of sync after pull                    | `pnpm migrate` (or restart the backend container — it runs `alembic upgrade head` on boot). |
| Want a clean slate                                   | `docker compose down -v` (drops the Postgres volume).               |
